Firefly Sprint1

Introduction

Goal: definere un modello qak di un sistema firefly che simula il comportamento di una lucciola.

Requirements

  1. Simulare con un modello qak il comportamento di una lucciola
  2. Estendere il modello per 3 e poi 100 lucciole.
  3. Per visualizzare le lucciole utilizzare la griglia canvas sviluppata in conway26GuiAlonePactorTcp

Requirement and problem analysis

In questo Sprint ci concentreremo solo sul requisito R1, in uno Sprint successivo verrà presa in considerazione l'esensione a più di una lucciola e poi si procederà verso la sincronizzazione del lampeggiamento.

Per comportamento di una lucciola, in questa fase del progetto, si intende che si accende e si spegne per un periodo di tempo random compreso tra 1 e 5 secondi. Una lucciola non interagisce con il mondo esterno, ma per propria decisione interna passa da accesa a spenta.
Una lucciola sarà quindi un QAttore proattivo, che dopo uno stato iniziale, alternerà tra due stati "accesa" e "spenta" dove resterà per un tempo random, invia un dispatch che informa dello stato in cui si trova. Facendo rapid prototyping, con un QAttore outputmock ho testato velocemente il funzionamento di questo comportamento. 

System fireflyquak
/*
 * il sistema firefly simula il comportamento di una lucciola
 */
 
// definizione dei messaggi
Dispatch luce : arg(V)		

// definizione del contesto
Context ctxfirefly ip [ host="localhost" port=8010 ]

// definizione degli attori
QActor firefly context ctxfirefly {
	State s0 initial {
		println("$name | starts") color cyan
	}
	Goto accesa
		
	State accesa {
		println("$name | accesa") color green
		forward outputmock -m luce : arg(1)
		[#
			val randomNumber = (1000..5000).random()
			delay(randomNumber.toLong())
		#]	
	}
	Goto spenta
	
	State spenta {
		println("$name | spenta") color red
		forward outputmock -m luce : arg(0)
		[#
			val randomNumber = (1000..5000).random()
			delay(randomNumber.toLong())
		#]
	}
	Goto accesa
	
	/*
	 * ATTORE PROATTIVO:
	 * con i due goto da accesa a spenta e viceversa si ha un ciclo infinito di vita della lucciola
	 * in cui passa da accesa a spenta e resta tale per un numero random compreso tra 1 e 5 secondi
	 * 
	 */
}

QActor outputmock context ctxfirefly {
	State s0 initial{
		println("$name | starts") color magenta
	}
	
	Transition t0 
		whenMsg luce -> handleLuce
		
	State handleLuce {
		printCurrentMessage color magenta
	}
	
	Transition t0 
		whenMsg luce -> handleLuce
}
