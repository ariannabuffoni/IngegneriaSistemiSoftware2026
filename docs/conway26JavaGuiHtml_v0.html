ConwayLife Sprint1

Introduction

Goal: Realizzazione in Java del GAME OF LIFE DI CONWAY con GUI del gioco realizzata come una pagina Web

Requirements

  1. Realizzare una versione in Java del gioco Life di Conway, come gioco zero-player.
    Il gioco consiste nell’introdurre una Griglia di Celle il cui stato (cella ‘viva’ o cella ‘morta’) evolve come stabilito dalle regole di ConwayLife.
  2. L’utente umano deve poter:
    • specificare la configurazione iniziale della griglia del gioco
    • vedere l’evoluzione del gioco in forma opportuna (si veda Problema della vista del gioco )
    • fermare e far ripartire l’evoluzione del gioco
    • pulire (a gioco fermo) la configurazione della griglia del gioco
  3. Realizzare una GUI realizata con una pagina Web e IoJavalin integrato nell'applicazione.

Problem analysis

Analizzo l'architettura del sistema che voglio realizzare:

A livello strutturale, oltre alle classi già realizzate, noto che IoJavalin dovrà implementare l'interfaccia IOutDev già definita e sarà lei a comunicare con la pagina html sul browser.
Sarà necessario un metodo setController per andare ad iniettare dentro a IoJavalin un riferimento al LifeController

A livello di interazione, è necessario andare a definire un protocollo di interazione tra IoJavalin e la pagina web, in modo che i messaggi utilizzino lo stesso linguaggio e che quindi entrambi possano essere in grado di interpretare correttamente il payload.
Utilizzerò dei messaggi che seguono la struttura dettata da IApplMessage , ovvero qualcosa del tipo msg( MSGID, MSGTYPE, SENDER, RECEIVER, CONTENT, SEQNUM ).
Andando a discriminare in base al contenuto del messaggio posso decidere quali operazioni far eseguire a IoJavalin. In particolare, i tipi di messaggi che il client può inviare (con relative azioni da parte del server):

  • msg( eval, dispatch, SENDER, lifectrl, start, 0 ), inviato quando il cliente clicca sul bottone "START" e farà partire il gioco, ovvero il server chiamerà la funzione onStart() di LifeController
  • msg( eval, dispatch, SENDER, lifectrl, stop, 0 ), inviato quando il cliente clicca sul bottone "STOP" e farà fermare il gioco, ovvero il server chiamerà la funzione onStop() di LifeController
  • msg( eval, dispatch, SENDER, lifectrl, clear, 0 ), inviato quando il cliente clicca sul bottone "CLEAR" e pulirà la griglia il gioco, ovvero il server chiamerà la funzione onClear() di LifeController
  • msg( eval, dispatch, SENDER, lifectrl, exit, 0 ), inviato quando il cliente clicca sul bottone "EXIT" e farà interrompere la connessione, ovvero il server chiamerà la propria funzione close()
  • msg( eval, dispatch, SENDER, lifectrl, cell(x,y), 0 ), inviato quando il cliente clicca su una cella (riporterà quindi x e y relativi) e ne cambierà lo stato, ovvero il server chiamerà la funzione switchCellStatus(x,y) di LifeController
Il server potrà inviare messaggi al client per comunicare la nuova configurazione della griglia o altri messaggi attraverso i metodi display.
